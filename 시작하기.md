## 다음 단계

### 1단계: Git 저장소 확인

Git Bash 또는 PowerShell에서 실행:

```bash
# 현재 디렉토리 확인
pwd

# Git 저장소 상태 확인
git status

# 원격 저장소 확인
git remote -v
```

원격 저장소가 없다면:
```bash
git remote add origin https://github.com/kwkimd/community_crawling.git
```

### 2단계: GitHub에 푸시

#### 방법 1: 자동 스크립트 사용 (추천)

Windows:
```bash
deploy.bat
```

Linux/Mac:
```bash
chmod +x deploy.sh
./deploy.sh
```

#### 방법 2: 수동 푸시

```bash
# 모든 파일 추가
git add .

# 커밋
git commit -m "Add deployment configuration for cloud hosting"

# 푸시
git push origin main
```

### 3단계: 클라우드 플랫폼 선택

#### 옵션 A: Render (무료, 추천)

1. https://render.com 접속
2. GitHub 계정으로 로그인
3. "New +" → "Web Service"
4. 저장소 선택: `kwkimd/community_crawling`
5. 설정:
   - Name: `community-dashboard`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python dashboard_server.py`
6. 환경 변수 추가:
   - `GEMINI_API_KEY`: (API 키 입력)
   - `GEMINI_MODEL`: `gemini-2.0-flash`
7. "Create Web Service" 클릭

#### 옵션 B: Railway (무료, 간단)

1. https://railway.app 접속
2. GitHub 계정으로 로그인
3. "New Project" → "Deploy from GitHub repo"
4. 저장소 선택: `kwkimd/community_crawling`
5. Variables 탭에서 환경 변수 추가
6. 자동 배포 시작

#### 옵션 C: Heroku (유료)

```bash
# Heroku CLI 설치 후
heroku login
heroku create community-dashboard
heroku config:set GEMINI_API_KEY=your_api_key
heroku config:set GEMINI_MODEL=gemini-2.0-flash
git push heroku main
```

### 4단계: 배포 확인

배포 완료 후:

```bash
# 헬스체크
curl https://your-app-url.com/health
```

브라우저에서 접속:
- 메인 페이지: `https://your-app-url.com/`
- API 문서: `https://your-app-url.com/docs`

## 필요한 것

## 도움말

### 문제 해결

**Q: Git 푸시가 안 돼요**
```bash
# 원격 저장소 확인
git remote -v

# 없으면 추가
git remote add origin https://github.com/kwkimd/community_crawling.git

# 브랜치 확인
git branch

# main 브랜치가 아니면
git checkout -b main
```

**Q: 빌드가 실패해요**
- `requirements.txt` 파일 확인
- Python 버전 확인 (3.11 권장)
- 로그에서 오류 메시지 확인

**Q: 서버가 시작 안 돼요**
- 환경 변수 설정 확인
- `GEMINI_API_KEY` 입력 확인
- 로그에서 오류 메시지 확인

**Q: 데이터가 사라져요**
- 클라우드 배포 시 SQLite는 휘발성입니다
- 서버 재시작 시 데이터 초기화됩니다
- 영구 저장이 필요하면 PostgreSQL 사용 권장
