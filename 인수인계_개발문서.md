## 아프니까사장이다 카페 분석 도구 (네이버 카페 크롤러 + AI 분석)

> **작성일**: 2026-03-09
> **현재 상태**: 전체 기능 구현 완료, 정상 동작 확인됨

## 1. 프로젝트 개요

네이버 카페 게시글을 자동 수집하고, Gemini AI로 분석하는 웹 기반 도구.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 자동 수집 | Playwright 브라우저로 네이버 카페 게시글·댓글 수집 |
| 다중 카페 | URL만 바꾸면 어떤 네이버 카페든 수집 가능 |
| DB 자동 저장 | 수집 완료 시 SQLite DB에 자동 저장 (중복 제외) |
| 게시글 목록 | 키워드/카테고리/카페명/날짜 필터링, 모달 상세보기 |
| Excel 다운로드 | 필터 조건 적용된 .xls 파일 다운로드 (댓글 포함) |
| AI 분석 | Gemini로 감성분석·키워드추출·트렌드분석·종합리포트 생성 |

## 5. DB 스키마

### posts 테이블
```sql
CREATE TABLE posts (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    title         TEXT NOT NULL,
    content       TEXT NOT NULL,
    comments      TEXT DEFAULT '[]',        -- JSON 배열: ["댓글1", "댓글2", ...]
    category      TEXT DEFAULT '일반',
    post_date     TEXT,                     -- YYYY-MM-DD 형식
    author        TEXT,
    created_at    TEXT DEFAULT (datetime('now', 'localtime')),
    view_count    INTEGER DEFAULT 0,
    comment_count INTEGER DEFAULT 0,
    site          TEXT DEFAULT '네이버 카페',
    cafe_name     TEXT DEFAULT '',           -- 구분용 카페 이름
    post_url      TEXT DEFAULT ''           -- 원본 게시글 URL
);
```

### analyses 테이블
```sql
CREATE TABLE analyses (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    created_at    TEXT DEFAULT (datetime('now', 'localtime')),
    analysis_type TEXT NOT NULL,            -- sentiment / keywords / trends / report
    post_ids      TEXT,                     -- JSON 배열: [1, 2, 3, ...]
    result        TEXT NOT NULL             -- JSON 문자열
);
```

### 인덱스
- `idx_posts_title_date` — 중복 체크 (title + post_date)
- `idx_posts_post_date`  — 날짜 정렬/필터
- `idx_posts_category`   — 카테고리 필터
- `idx_posts_cafe_name`  — 카페명 필터 (**마이그레이션 완료 후 생성**)

## 7. 크롤러 동작 방식 (scraper.py)

### 수집 흐름
```
start_scraping() 호출
  → asyncio.create_task(_do_scrape())
    → board_url 또는 기본 BOARD_URL로 이동
    → _collect_article_links(): 게시글 링크 + 날짜 수집 (멀티 페이지)
    → 날짜 필터 적용
    → 개별 _scrape_article() 호출 (새 탭에서 열기)
      → iframe 탐지 (네이버 카페 구조)
      → 제목, 본문, 날짜, 작성자, 조회수, 댓글 추출
    → db.create_post() 자동 저장
```

### 핵심 상수 (기본값)
```python
CAFE_URL  = "https://cafe.naver.com/jihosoccer123"
BOARD_URL = "https://cafe.naver.com/f-e/cafes/23611966/menus/0?q=..."
CAFE_ID   = "23611966"
```
> UI에서 게시판 URL을 입력하면 BOARD_URL 대신 사용됨 (동적 지원)

### 카페명 자동 추출 (미입력 시)
```python
m = re.search(r"cafe\.naver\.com/(?:f-e/cafes/\d+|([^/?#]+))", base_url)
resolved_cafe_name = m.group(1) if (m and m.group(1)) else "네이버 카페"
```

### 게시글 링크 수집 JS 정규식 (범용)
```javascript
/cafe\.naver\.com\/[^\/\?#]+\/\d+/.test(href)
// 이전에는 jihosoccer123 하드코딩 → 현재 범용 패턴으로 변경됨
```

## 9. 해결한 주요 버그

### [버그 1] cafe_name 인덱스 순서 오류 (가장 최근)
- **오류**: `sqlite3.OperationalError: no such column: cafe_name`
- **원인**: `executescript()` 안에 `CREATE INDEX ... ON posts (cafe_name)` 포함 → 기존 DB에 컬럼이 없는 상태에서 인덱스 생성 시도
- **해결**: 인덱스 생성을 `executescript()` 밖으로 꺼내 마이그레이션(`ALTER TABLE`) 완료 후 실행

### [버그 2] 날짜 잘못 파싱 (카페 개설일 2011-09-24)
- **원인**: body 텍스트에서 카페 개설일을 게시일로 파싱
- **해결**: body 폴백 파싱 시 2019년 이전 날짜 무시

### [버그 3] 서버 즉시 종료
- **원인**: `main.py` 하단에 `uvicorn.run()` 블록 누락
- **해결**: `if __name__ == "__main__": uvicorn.run(...)` 추가

## 10. Gemini 분석 역할 컨텍스트

`gemini_analyzer.py`의 `ROLE_CONTEXT`:
- 배달플랫폼 서비스 모니터링 담당자 관점
- 자영업자 매출 증대 + 플랫폼 수익 성장 동시 달성
- 정부 정책/법 의존 해결책 제외, 플랫폼 자체 실행 가능한 것만

분석 유형 4종:
1. **감성 분석** — 긍정/부정/중립 비율, 이탈 위험도
2. **키워드 추출** — 상위 키워드, 플랫폼 개선 연결 포인트
3. **트렌드 분석** — 시간 흐름별 주제 변화, 신규 니즈
4. **종합 리포트** — 핵심 발견사항, 우선순위별 개선안

## 12. 다른 네이버 카페 수집 방법

1. 수집할 카페 게시판 URL 복사
   - 예: `https://cafe.naver.com/f-e/cafes/[카페ID]/menus/[메뉴ID]?...`
2. UI → **자동 수집 탭** → **게시판 URL** 칸에 붙여넣기
3. **카페명** 칸에 구분용 이름 입력 (예: `맘카페`, `자영업자모임`)
4. 날짜 범위 설정 → **수집 시작**
5. 게시글 목록에서 **전체 카페** 드롭다운으로 카페별 필터링

## 14. 주의사항

1. **로그인 필수**: 네이버 카페는 로그인 없이 수집 불가. 브라우저 열기 후 수동 로그인 후 수집 시작
2. **봇 감지**: 빠른 수집 시 네이버 봇 감지 가능. 현재 1.2~2.5초 랜덤 딜레이 적용 중
3. **DB 파일 보관**: `cafe_posts.db` 삭제 시 전체 데이터 소실. 정기 백업 권장
4. **API 키 보안**: `.env` 파일의 `GEMINI_API_KEY`는 외부 유출 금지
5. **xlwt 제한**: xlwt는 `.xls`(구형 Excel 포맷)만 지원. `.xlsx` 필요 시 `openpyxl`로 교체 필요
