# 배포 체크리스트

## 배포 전 확인사항

### 1. 파일 확인
- [ ] `dashboard_server.py` - 대시보드 서버 코드
- [ ] `database.py` - 데이터베이스 관리
- [ ] `gemini_analyzer.py` - AI 분석 모듈
- [ ] `templates/dashboard.html` - 대시보드 UI
- [ ] `requirements.txt` - Python 패키지 목록
- [ ] `.env.example` - 환경 변수 예시
- [ ] `.gitignore` - Git 제외 파일 목록
- [ ] `README.md` - 프로젝트 설명
- [ ] `DEPLOYMENT.md` - 배포 가이드

### 2. 환경 변수 준비
- [ ] Gemini API 키 발급 완료
  - https://aistudio.google.com/app/apikey
- [ ] `.env` 파일이 `.gitignore`에 포함되어 있는지 확인
- [ ] `.env.example` 파일 내용 확인

### 3. 로컬 테스트
```bash
# 패키지 설치
pip install -r requirements.txt

# 환경 변수 설정
cp .env.example .env
# .env 파일에 실제 API 키 입력

# 서버 실행
python dashboard_server.py

# 브라우저에서 테스트
# http://localhost:8000
```

- [ ] 서버가 정상적으로 시작되는지 확인
- [ ] 대시보드 페이지가 로드되는지 확인
- [ ] `/health` 엔드포인트 응답 확인

## GitHub 푸시

### 1. Git 저장소 확인
```bash
# 원격 저장소 확인
git remote -v

# 없으면 추가
git remote add origin https://github.com/kwkimd/community_crawling.git
```

### 2. 푸시 실행
```bash
# Windows
deploy.bat

# Linux/Mac
chmod +x deploy.sh
./deploy.sh
```

또는 수동으로:
```bash
git add .
git commit -m "Deploy dashboard to GitHub"
git push origin main
```

- [ ] GitHub 저장소에 파일이 업로드되었는지 확인
- [ ] `.env` 파일이 업로드되지 않았는지 확인

## 클라우드 배포

### Render 배포 (추천)

1. [ ] https://render.com 접속 및 로그인
2. [ ] "New +" → "Web Service" 선택
3. [ ] GitHub 저장소 연결: `kwkimd/community_crawling`
4. [ ] 설정 입력:
   - Name: `community-dashboard`
   - Environment: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python dashboard_server.py`
5. [ ] 환경 변수 추가:
   - `GEMINI_API_KEY`: (실제 API 키)
   - `GEMINI_MODEL`: `gemini-2.0-flash`
6. [ ] "Create Web Service" 클릭
7. [ ] 빌드 로그 확인
8. [ ] 배포 완료 후 URL 접속 테스트

### Railway 배포

1. [ ] https://railway.app 접속 및 로그인
2. [ ] "New Project" → "Deploy from GitHub repo"
3. [ ] 저장소 선택: `kwkimd/community_crawling`
4. [ ] Variables 탭에서 환경 변수 추가:
   - `GEMINI_API_KEY`: (실제 API 키)
   - `GEMINI_MODEL`: `gemini-2.0-flash`
5. [ ] 자동 배포 시작 확인
6. [ ] Settings → Networking에서 도메인 생성
7. [ ] 생성된 URL 접속 테스트
