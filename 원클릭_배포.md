# 🚀 원클릭 배포

## GitHub에서 바로 배포하기

### 방법 1: Deploy to Render 버튼 (가장 쉬움)

1. GitHub 저장소로 이동:
   ```
   https://github.com/kwkimd/community_crawling
   ```

2. README.md에서 "Deploy to Render" 버튼 클릭

3. Render 로그인 (GitHub 계정 사용)

4. 환경 변수 입력:
   - `GEMINI_API_KEY`: API 키 입력

5. "Apply" 클릭

6. 완료! 🎉

---

### 방법 2: Render 대시보드에서 직접

1. https://dashboard.render.com 접속

2. "New +" → "Web Service"

3. GitHub 저장소 연결:
   ```
   kwkimd/community_crawling
   ```

4. 자동으로 `render.yaml` 설정 감지

5. 환경 변수 입력:
   - `GEMINI_API_KEY`: API 키 입력

6. "Create Web Service" 클릭

7. 완료! 🎉

---

### 방법 3: Railway (대안)

1. https://railway.app 접속

2. "New Project" → "Deploy from GitHub repo"

3. 저장소 선택:
   ```
   kwkimd/community_crawling
   ```

4. Variables 탭에서 환경 변수 추가:
   ```
   GEMINI_API_KEY=your_api_key_here
   GEMINI_MODEL=gemini-3-flash-preview
   ```

5. 자동 배포 시작

6. 완료! 🎉

---

## 배포 후 확인

### 1. URL 확인
Render 또는 Railway에서 제공하는 URL:
```
https://community-dashboard-xxxx.onrender.com
```

### 2. 헬스체크
```bash
curl https://your-url.com/health
```

예상 응답:
```json
{
  "status": "healthy",
  "service": "dashboard",
  "total_posts": 0
}
```

### 3. 브라우저 접속
- 메인 페이지: `https://your-url.com/`
- API 문서: `https://your-url.com/docs`

## 자동 배포 설정

배포 완료 후, GitHub에 푸시하면 자동으로 재배포됩니다:

```bash
git add .
git commit -m "Update code"
git push origin main
```

→ Render/Railway가 자동으로 감지하고 배포!
