# 웹 크롤링 작동 원리 설명서
*팀 내 공유용 - 비개발자도 이해하기 쉬운 설명*

---

## 🤖 크롤링이란?

**사람이 웹사이트를 보는 것처럼 컴퓨터가 자동으로 웹페이지를 방문해서 정보를 수집하는 기술**

웹 크롤링은 반복적이고 시간이 많이 걸리는 데이터 수집 작업을 자동화하는 방법입니다.

---

## 📱 일반적인 웹 이용 vs 크롤링 비교

### 👤 사람이 하는 일:
1. 브라우저 열기 (크롬, 엣지 등)
2. 네이버 카페 접속
3. 로그인하기
4. 게시판 들어가기
5. 게시글 하나씩 클릭해서 읽기
6. 제목, 내용, 작성자 등을 눈으로 보고 기억하거나 메모
7. 다음 게시글로 이동해서 반복

### 🤖 크롤링 프로그램이 하는 일:
1. **자동으로 브라우저 열기** (사람 눈에 보이는 실제 브라우저)
2. **자동으로 네이버 카페 접속**
3. **사람이 로그인** (보안상 사람이 직접 해야 함)
4. **자동으로 게시판 페이지 이동**
5. **자동으로 게시글 링크들 수집** (1페이지, 2페이지... 계속)
6. **자동으로 각 게시글 클릭해서 들어가기**
7. **자동으로 정보 추출** (제목, 내용, 작성자, 날짜, 댓글 등)
8. **자동으로 데이터베이스에 저장**

---

## 🔍 우리 시스템의 특별한 점

### 1. 실제 브라우저 사용
- 가짜 요청이 아닌 **진짜 크롬 브라우저를 자동 조작**
- 네이버가 봤을 때 사람이 사용하는 것과 동일
- 자연스러운 웹 브라우징 패턴 구현

### 2. 지능형 자동 분류
수집한 데이터를 자동으로 분석해서 분류합니다:

**감성 분석**
- "최악이다", "미친" → 부정
- "좋다", "만족" → 긍정  
- 중립적 표현 → 중립

**주체 구분**
- "우리 가게", "매장" → 업주
- "배달왔는데", "라이더" → 라이더
- 일반적 불만 → 소비자

**위험도 계산**
- 부정적 키워드 많으면 → 높은 위험도 (2단계)
- 일반적 불만 → 보통 위험도 (1단계)
- 긍정적 내용 → 낮은 위험도 (0단계)

**서비스 구분**
- "쿠팡이츠" 언급 → 쿠팡이츠
- "요기요" 언급 → 요기요
- 기본값 → 배달의민족

### 3. 중복 방지 시스템
- 이미 수집한 게시글은 다시 저장하지 않음
- 제목 + 작성날짜로 중복 판단
- 데이터베이스 용량 절약 및 정확성 보장

## ⚡ 속도와 효율성 비교

### 👤 사람이 수작업으로 할 경우:
- 게시글 1개 읽고 정리: **약 30초~1분**
- 250개 게시글 처리: **약 2~4시간**
- 분류 작업까지 포함: **약 4~6시간**

### 🤖 크롤링 프로그램:
- 게시글 1개 수집 및 분류: **약 2~3초**
- 250개 게시글 처리: **약 10~15분**
- **전체 작업 시간: 약 10~20배 단축**

---

## 🛡️ 안전장치 및 윤리적 고려사항

### 1. 속도 조절
- 게시글 사이에 **1~3초 대기**
- 서버 부하 방지 및 차단 위험 최소화
- 자연스러운 사용 패턴 구현

### 2. 사람처럼 행동
- 실제 브라우저 사용으로 자연스러운 접근
- 마우스 클릭, 스크롤 등 실제 사용자 동작 모방
- 로그인은 반드시 사람이 직접 수행

### 3. 에러 처리
- 페이지 로딩 실패시 **자동 재시도**
- 접근 불가능한 게시글은 **건너뛰기**
- 예외 상황 발생시 **안전하게 중단**

## 📊 최종 결과물

### CSV 형식 데이터 (30개 컬럼 구조)

**기본 정보**
- 제목, 내용, 작성자, 작성날짜
- 조회수, 댓글수, 게시글 URL

**자동 분류 결과**
- 감성구분 (긍정/부정/중립)
- 위험도 (0~2단계)
- 주체구분 (업주/라이더/소비자/기자)
- 서비스구분 (배달의민족/쿠팡이츠/요기요)

**분석 결과**
- 자동 요약
- 키워드 추출
- 분류체계 (대분류/중분류/소분류)
- 리스크 분류

### Excel 파일 내보내기
- **바로 분석 가능한 형태**로 제공
- 필터링, 정렬, 차트 생성 등 자유롭게 활용
- 기존 분석 도구와 호환

## 💡 비즈니스 가치

### 1. 시간 절약
- **수작업 대비 90% 시간 단축**
- 반복 작업 자동화로 인력 효율성 극대화

### 2. 정확성 향상
- **사람 실수 없이 일관된 기준**으로 분류
- 객관적이고 표준화된 데이터 수집

### 3. 확장성
- **여러 카페, 커뮤니티 동시 모니터링** 가능
- 수집 대상 쉽게 추가/변경 가능

### 4. 실시간성
- **새 게시글 자동 감지** 및 수집
- 24시간 무인 모니터링 가능

### 5. 데이터 품질
- **구조화된 데이터**로 즉시 분석 가능
- 일관된 형태로 장기간 데이터 축적
